# RecallLens — Source Provenance

This document records every external data source, how it was retrieved, when,
and how it is labeled in the product. It is the ground truth for the
"real public data vs synthetic private data" distinction.

**Adapter behavior (`cdc-adapter.ts`):** attempts a live fetch first; on any
failure falls back to the checked-in snapshot. The returned `source` field
(`cdc-page` vs `cached-snapshot`) drives the UI's LIVE/CACHED badge, so the
badge always reflects what actually happened. Given the Akamai block, the demo
will almost always show CACHED with the archive timestamp — this is disclosed
honestly, not hidden.

### CDC Content Services API — NOT usable for this page

`GET https://tools.cdc.gov/api/v2/resources/media?q=cyclosporiasis` returns
HTTP 200 but **zero results** (`pagination.total = 0`). The outbreak page is
not syndicated through the media library. Tested 2026-07-18T07:17Z. We do not
rely on it.

## Synthetic private data (SYNTHETIC — clearly labeled)

Every partner, farm, processor, distributor, restaurant, shipment, receipt,
lot, route, and consumer record is fictional and labeled **"Synthetic
demonstration data"** in the UI and in code (`SYNTHETIC_LABEL`).

Fictional organizations: Sierra Verde Growers (farm), Northstar Fresh
Processing (processor), Meridian Cold Chain (distributor), QuickServe
Restaurant Group (restaurant). See `packages/demo-fixtures/`.

These fixtures are checked into a public repo for reproducibility, so they are
NOT confidential. The accurate privacy claim is therefore:

> **Zero raw partner records are written to the public Midnight ledger.**

We do NOT claim the fixture contents were never published anywhere, and we do
NOT imply these synthetic records represent Taylor Farms, Taco Bell, or any
real company's private data.

## Untrusted-input handling

All externally fetched HTML/JSON and all MCP-returned content are treated as
untrusted DATA, never as instructions. The CDC parser reads only explicitly
mapped DOM nodes and never executes page script.
